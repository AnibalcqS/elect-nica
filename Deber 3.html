Programación Orientada a Objetos

Entendiendo el paradigma basado en Clases y Objetos

1. Clases y Objetos

Imagina una Clase como el plano arquitectónico de una casa. No es una casa real, sino el diseño. Un Objeto es la casa construida a partir de ese plano.

  • Clase: El molde o plantilla (ej: Persona).
  • Objeto: La instancia real (ej: Juan, María).

2. Los 4 Pilares de la POO

Encapsulamiento

Oculta los detalles internos y protege los datos, exponiendo solo lo necesario.

Abstracción

Simplifica modelos complejos mostrando solo las características esenciales.

Herencia

Permite que una clase nueva tome propiedades de una clase ya existente.

Polimorfismo

Capacidad de que diferentes objetos respondan al mismo mensaje de formas distintas.

3. Ejemplo de Código

class Vehiculo {
    constructor(marca) {
        this.marca = marca;
    }
    arrancar() {
        return `El ${this.marca} está encendido.`;
    }
}

// Herencia
class Auto extends Vehiculo {
    tocarClaxon() {
        return "¡Beep beep!";
    }
}
